# React Best Practices 체크리스트

Vercel의 React Best Practices를 기반으로 한 프로젝트 가이드라인

## CRITICAL (최우선)

### 1. 비동기 워터폴 제거
- [ ] 병렬 처리 가능한 API 호출은 `Promise.all()` 사용
- [ ] 조건부로 필요한 데이터만 로드 (조건 확인 후 fetch)
- [ ] 서버 컴포넌트에서 데이터 페칭 최적화

### 2. 번들 크기 최적화
- [ ] 동적 import 사용 (`React.lazy`, `next/dynamic`)
- [ ] 불필요한 라이브러리 제거
- [ ] Tree-shaking 가능한 import 방식 사용

## HIGH (높은 우선순위)

### 3. 서버 사이드 성능
- [ ] 정적 생성 가능한 페이지는 `generateStaticParams` 사용
- [ ] ISR(Incremental Static Regeneration) 적절히 활용
- [ ] Edge Runtime 고려 (가능한 경우)

### 4. 클라이언트 사이드 데이터 페칭
- [ ] `useSWR` 또는 `React Query` 사용 (필요시)
- [ ] 캐싱 전략 수립
- [ ] 에러 바운더리 구현

### 5. 리렌더 최적화
- [ ] `React.memo` 적절히 사용
- [ ] `useMemo`로 무거운 계산 캐싱
- [ ] `useCallback`으로 함수 참조 안정화
- [ ] 상태를 필요한 만큼만 분리

## MEDIUM (중간 우선순위)

### 6. 렌더링 성능
- [ ] 가상화(Virtualization) 고려 (긴 리스트)
- [ ] 이미지 최적화 (`next/image` 사용)
- [ ] 폰트 최적화 (`next/font` 사용)

### 7. 고급 패턴
- [ ] Lazy State Initialization 사용 ✅ (이미 적용됨)
- [ ] Context API 최적화 (Provider 분리)
- [ ] Compound Components 패턴 고려

## LOW (낮은 우선순위)

### 8. JavaScript 성능
- [ ] 불필요한 루프 최적화
- [ ] 메모이제이션 전략 검토
- [ ] 디바운싱/스로틀링 적절히 사용

## 현재 프로젝트 적용 상태

### ✅ 이미 적용됨
- Lazy State Initialization (`useState(() => ...)`)
- `useMemo`로 필터링 로직 최적화
- `useCallback`으로 핸들러 최적화
- `key` prop으로 컴포넌트 리마운트 제어

### 🔄 개선 필요
- 코드 스플리팅 (GameMode 컴포넌트)
- 번들 분석 도구 추가
- 성능 모니터링 설정
